Detailed Description

A Runner runs tests and notifies a org.junit.runner.notification.RunNotifier of significant events as it does so. You will need to subclass Runner when using org.junit.runner.RunWith to invoke a custom runner. When creating a custom runner, in addition to implementing the abstract methods here you must also provide a constructor that takes as an argument the Class containing the tests.

The default runner implementation guarantees that the instances of the test case class will be constructed immediately before running the test and that the runner will retain no reference to the test case instances, generally making them available for garbage collection.
