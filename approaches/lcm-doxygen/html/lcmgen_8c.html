The LCM grammar is implemented here with a recursive-descent parser. handle_file is the top-level function, which calls parse_struct/parse_enum, and so on.

Every LCM type has an associated "signature", which is a hash of various components of its delcaration. If the declaration of a signature changes, the hash changes with high probability.

Note that this implementation is sloppy about memory allocation: we don't worry about freeing memory since the program will exit after parsing anyway.
