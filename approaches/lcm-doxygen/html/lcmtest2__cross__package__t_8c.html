Sets the queue capacity for a subscription. Some LCM providers (e.g., the default multicast provider) are implemented using a background receive thread that constantly revceives messages from the network. As these messages are received, they are buffered on per-subscription queues until dispatched by lcm_handle(). This function how many messages are queued before dropping messages.

Parameters

num_messagesThe maximum number of messages to queue on the subscription.
